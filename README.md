## 选题一：AI Agent 框架搭建——从"会说话"到"能做事"

### 1. 任务目标

从零搭建简化版 AI Agent 框架，实现：
> 接收自然语言指令 → 自主选择工具 → 调用工具获取结果 → 整理回复用户

支持多轮"思考-行动"循环。

### 2. 技术栈

- Python + 任意 LLM API（OpenAI / DeepSeek / 通义千问）或本地模型（如 Qwen2.5-3B）
- **禁止**使用 LangChain、LlamaIndex 等高层框架

### 3. 核心任务拆解

#### 阶段一：环境准备与原理验证（第 1 周）

| 任务 | 详细内容 |
|------|----------|
| API 接入 | 申请 LLM API Key（推荐 DeepSeek，有免费额度），跑通第一个 API 调用 |
| ReAct 范式学习 | 理解 Reasoning + Acting 循环：模型交替输出"思考"和"动作" |
| Prompt 设计 | 编写系统提示词（System Prompt），约束模型输出格式为 JSON |
| 输出格式定义 | 定义工具调用的 JSON schema：`{"action": "tool_name", "params": {...}}`，以及最终回复格式 `{"action": "final_answer", "content": "..."}` |
| 调研产出 | 开题报告中写清楚：ReAct vs OpenAI Function Calling 的区别，Prompt 约束策略 |

#### 阶段二：核心功能开发（第 2 周）

| 任务 | 详细内容 |
|------|----------|
| 工具系统 | 实现至少 3 个工具：**计算器**（eval/math 表达式解析）、**维基百科搜索**（爬虫或 wikipedia-api）、**本地文件读写**（open/read/write） |
| 解析器 | 解析 LLM 返回的 JSON，提取 action 和参数，健壮处理 JSON 解析错误 |
| 执行引擎 | 根据解析结果调用对应工具函数，获取返回值 |
| 对话循环 | 实现主循环：用户输入 → LLM 输出 → 解析 → 执行工具 → 结果喂回 LLM → 重复直到返回 final_answer |
| 对话历史管理 | 维护 `messages` 列表，管理上下文窗口（超出 token 限制时自动截断或摘要） |

#### 阶段三：优化与实验（第 3 周）

| 任务 | 详细内容 |
|------|----------|
| 多步推理测试 | 构造需要多步调用的任务（如"查爱因斯坦出生年份，算他活了多少岁"），验证 Agent 能正确串联多个工具 |
| 边界情况处理 | 工具调用失败重试、JSON 格式错误的 fallback 策略、超时处理 |
| 实验记录 | 截取多步推理的对话日志，标注每步的思考和动作 |

#### 阶段四：结项与交付（第 4 周）

| 任务 | 详细内容 |
|------|----------|
| 实验报告 | ReAct Prompt 模板展示、Agent 决策闭环流程图、多步推理对话日志分析 |
| 演示视频 | 录制从输入指令到多步完成的全过程 |
| PPT | 重点讲核心原理和算法实现 |

### 4. 进阶加分项

| 方向 | 说明 |
|------|------|
| 流式输出 | 实现 SSE / streaming，逐步展示 LLM 的思考过程 |
| 记忆系统 | 短期记忆（对话摘要）+ 长期记忆（向量数据库存重要信息） |
| Web 界面 | 用 Gradio 或 Streamlit 搭建交互界面 |
| 多 Agent 协作 | 规划 Agent + 执行 Agent 的分工模式 |

### 5. 学到的核心能力

- LLM 应用层核心设计模式
- "提示词即程序"思维
- API 调用与 JSON 解析基本功
- ReAct 范式工程落地

## 选题二：NumPy 实现神经网络——揭开自动微分的"黑盒"

### 1. 任务目标

只使用 NumPy，从零实现多层感知机（MLP），在 MNIST 上训练推理，测试集准确率 ≥ 95%。

### 2. 技术栈

- Python + NumPy（核心）+ Matplotlib（画曲线）
- **禁止**使用 PyTorch、TensorFlow 等自动微分框架
- 可以从 `torchvision.datasets` 加载 MNIST 数据（仅用于数据读取）

### 3. 核心任务拆解

#### 阶段一：环境准备与原理验证（第 1 周）

| 任务 | 详细内容 |
|------|----------|
| 数学推导 | 在纸上/LaTeX 中推导全连接层的前向传播、反向传播梯度公式。链式法则如何体现在每一层？Weight 和 Bias 的梯度如何从后一层传过来？ |
| 数据加载 | 下载 MNIST，用 NumPy 加载为 numpy 数组，归一化到 [0,1] 或 [-1,1] |
| 基础组件 | 实现 Layer 基类（forward + backward 接口）、Linear 层（W, b 初始化用 Xavier/He）、ReLU/Sigmoid/Softmax 激活函数 |
| 调研产出 | 对比不同激活函数的优劣（ReLU vs Sigmoid vs Tanh），不同初始化方法的效果 |

#### 阶段二：核心功能开发（第 2 周）

| 任务 | 详细内容 |
|------|----------|
| 前向传播 | 实现完整的前向传播链路：Input → Linear → ReLU → Linear → ReLU → Linear → Softmax |
| 损失函数 | 实现 CrossEntropyLoss（交叉熵）+ 可选 MSE |
| 反向传播 | **最硬核部分**：为每个 Layer 手动实现 backward()，用链式法则计算每个参数的梯度。特别注意 Softmax + CrossEntropy 的合并梯度（简化版公式） |
| 优化器 | 实现 SGD（带动量可选），进阶可做 Momentum、Adam |
| 训练循环 | mini-batch 划分（如 batch_size=64）、shuffle、多个 epoch |

#### 阶段三：优化与实验对比（第 3 周）

| 任务 | 详细内容 |
|------|----------|
| 调参实验 | 对比不同超参数的影响：学习率（0.1, 0.01, 0.001）、batch 大小（32, 64, 128）、网络深度（2层 vs 3层 vs 4层）、隐藏层宽度 |
| 正则化 | 过拟合时加 L2 正则（Weight Decay）或 Dropout |
| 学习率调整 | 实现学习率衰减策略（Step Decay / Exponential Decay） |
| 可视化 | 用 Matplotlib 绘制 Loss 曲线和 Accuracy 曲线（训练集 vs 验证集） |

#### 阶段四：结项与交付（第 4 周）

| 任务 | 详细内容 |
|------|----------|
| 最终训练 | 调优后跑最终版本，确保测试集准确率 ≥ 95% |
| 实验报告 | ① 反向传播数学推导（手写拍照或 LaTeX）② 各层 Matrix Shape 变换过程图 ③ Loss/Accuracy 曲线 ④ 超参数对比实验表格和曲线 ⑤ 代码模块结构说明（Layer / Loss / Optimizer 的类设计） |
| 演示视频 | 录制训练过程和推理演示 |
| PPT | 重点讲核心原理（前向/反向传播公式推导）和算法实现 |

### 4. 进阶加分项

| 方向 | 说明 |
|------|------|
| CNN | 实现卷积层（im2col 加速）和池化层，准确率冲到 99% |
| BatchNorm | 实现 Batch Normalization 层 |
| 其他数据集 | 用自己框架跑 Fashion-MNIST 或 CIFAR-10 |
| autograd 引擎 | 写一个简化版自动求导引擎（参考 micrograd），构建计算图 |
| 更多优化器 | 完整实现 Adam、RMSprop |

### 5. 学到的核心能力

- 彻底搞懂神经网络前向/反向传播的数学原理
- 矩阵求导与链式法则的工程落地
- 理解 PyTorch/TensorFlow 底层在做什么
- 为读论文和看框架源码打基础

## 选题三：下棋 AI 与博弈搜索——经典 AI 的智慧与魅力

### 1. 任务目标

实现能与人（或 AI）对战的棋类 AI。可选棋种：
- 井字棋（最简单，不推荐单独使用）
- **五子棋**（建议 8×8 或 9×9 小棋盘）
- 黑白棋 / Othello

AI 需在合理时间内（每步 ≤ 5 秒）走出有水平的棋。

### 2. 技术栈

- Python（开发快）或 C++（性能好）
- 图形界面：pygame（推荐）或纯命令行打印棋盘
- **禁止**调用现成棋类 AI 库

### 3. 核心任务拆解

#### 阶段一：环境准备与原理验证（第 1 周）

| 任务 | 详细内容 |
|------|----------|
| 选棋种 | 确定做五子棋 / 黑白棋 / 井字棋（建议五子棋 8×8，复杂度适中） |
| 棋盘表示 | 二维数组（numpy 或 list of list）表示棋盘状态，`0`=空, `1`=黑, `-1`=白 |
| 走法生成 | 获取当前玩家所有合法落子位置 |
| 胜负判定 | 检查横/竖/对角线是否有连续 N 子（五子棋 N=5，黑白棋终局比较数量） |
| 终局检测 | 棋盘满 / 无合法走法 / 一方连成五子 |
| Minimax 原型 | 在井字棋（状态空间小）上先验证 Minimax 算法的正确性与完整性 |
| 调研产出 | 当前主流评估函数设计思路，不同棋种的启发式策略 |

#### 阶段二：核心功能开发（第 2 周）

| 任务 | 详细内容 |
|------|----------|
| Minimax 完整实现 | 构建博弈树，假设对手最优，递归找到当前最优走法 |
| Alpha-Beta 剪枝 | 在 Minimax 基础上裁掉不可能影响结果的分支，搜索量降低一个数量级 |
| 评估函数 | 当搜索不到终局时，用启发式评估当前局面。五子棋例子：活三 +50、活四 +500、冲四 +100、眠三 +10、连五 +10000，可自由设计权重 |
| 迭代加深搜索 | 逐步递增搜索深度（depth=1,2,3,...），在时限到期时返回当前最优解，保证无论什么情况都有合理走法 |
| 命令行界面 | 打印棋盘，人输入坐标走棋 |

#### 阶段三：优化与实验对比（第 3 周）

| 任务 | 详细内容 |
|------|----------|
| 性能对比实验 | ① 纯 Minimax vs Alpha-Beta 在不同深度（depth=2,4,6,8）下的耗时对比 ② 搜索节点数对比 ③ 同等时限下的胜率对比 |
| 评估函数对比 | 多组不同权重配置的棋力对比（不同评估函数的 A vs B） |
| 走法排序优化 | 按历史启发对候选走法预排序，加速 Alpha-Beta 剪枝 |
| 图形界面 | 用 pygame 实现可点击的棋盘界面（可选，时间充裕做） |

#### 阶段四：结项与交付（第 4 周）

| 任务 | 详细内容 |
|------|----------|
| 实验报告 | ① Minimax 递归终止条件详述 + 分值权重表 ② 纯 Minimax vs Alpha-Beta 耗时/搜索节点数对比表 ③ 不同评估函数的棋力对比 ④ 不同搜索深度对棋力的影响 |
| 演示视频 | 人 vs AI 完整对局，展示 AI 思考过程和走法 |
| PPT | 重点讲 Minimax 算法原理、Alpha-Beta 剪枝为什么有效、评估函数设计思路 |

### 4. 进阶加分项

| 方向 | 说明 |
|------|------|
| 置换表 | Transposition Table 缓存已搜索过的局面，大幅减少重复计算 |
| 走法排序 | 杀手启发（Killer Heuristic）+ 历史启发（History Heuristic）进一步优化剪枝效率 |
| MCTS | 用蒙特卡洛树搜索替代 Minimax，实现 AlphaGo 的核心算法思路。四个步骤：Selection → Expansion → Simulation → Backpropagation |
| AI 天梯赛 | 做 AI vs AI 对战系统，让全班的 AI 互相切磋排名 |

### 5. 学到的核心能力

- 递归与回溯的工程化运用
- 状态空间搜索理解（AI 经典范式）
- 算法优化和性能调试能力
- 认识到"AI ≠ 深度学习"，经典搜索同样强大

## 三个选题对比总结

| 维度 | 选题一：AI Agent | 选题二：NumPy 神经网络 | 选题三：棋类 AI |
|------|:---:|:---:|:---:|
| 偏重方向 | LLM 应用 / 软件工程 | 深度学习数学原理 | 经典 AI / 算法搜索 |
| 最难部分 | Prompt 工程 + 解析鲁棒性 | 反向传播手动梯度推导 | 评估函数设计 + 剪枝优化 |
| 代码量 | 中等（500-800 行） | 中等偏多（800-1200 行） | 中等（500-800 行） |
| 数学要求 | 低 | **高**（矩阵求导、链式法则） | 中（递推、博弈论入门） |
| 成就感 | 做出来就能用，像 ChatGPT | 真懂了神经网络底层 | 能赢人的 AI，直观可见 |
| 适合人群 | 想做应用/产品方向 | 想深入理解深度学习 | 喜欢算法/逻辑的同学 |
| 必达标 | 3 工具 + 多步推理 | MNIST ≥ 95% | Minimax + Alpha-Beta |
